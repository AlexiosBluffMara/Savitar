# Gemma 4 Good Hackathon

This directory now reflects the active hackathon branch focus: a local-first Gemma 4 operator system for Chicago small businesses and educators, run from a Mac Mini and surfaced through invite-only Discord plus an operator web console.

## Official competition snapshot

- Deadline: May 18, 2026 at 11:59 PM UTC.
- Team limit: 5 people.
- Submission limit: 1 writeup per team.
- Required assets: public Kaggle writeup, public YouTube video of 3 minutes or less, public code repository, public live demo, and a media gallery with a cover image.
- Writeup cap: 1,500 words.
- Winners license: CC-BY 4.0.
- External data and tools are allowed only when they are reasonably accessible to other participants at minimal cost.

## Active submission thesis

Savitar should present as a serious, local-first operator for trusted community knowledge:

- Primary operator story: Alexios Bluff Mara LLC runs Savitar on a Mac Mini for Chicago small-business support, educator workflows, and invite-only neighborhood knowledge operations.
- Primary end-user surface: Discord with role-gated channels, files, links, and short video workflows.
- Primary technical story: Gemma 4 runs locally through Ollama on bare metal, with local markdown and document packs stored on attached SSD.
- Primary product differentiators: evidence-backed replies, operator review, local knowledge retrieval, and an educator-friendly Live Mode.

## Targeted award coverage

- Main Track: yes.
- Impact Track: optimize for Future of Education, Digital Equity & Inclusivity, and Safety & Trust.
- Special Technology Track: Ollama is the direct fit; Unsloth is the highest-upside secondary fit if a focused domain adapter ships in time.
- Stretch coverage: Cactus or LiteRT only if a truthful mobile-routing demo exists. Do not overclaim these.

The public rules explicitly allow a project to win both a Main Track prize and a Special Technology prize. Public organizer discussion has not yet clarified whether one entry can win multiple awards within the same track, so the docs should avoid assuming that outcome.

## Build posture

- Keep inference local through Ollama.
- Keep public claims honest about what is shipped versus planned.
- Use Discord and the operator dashboard as the main proof surface.
- Treat markdown packs, operator review, and provenance as the trust backbone.
- Use GitHub and Copilot for background leverage, not as a substitute for the local Gemma story.
