# Provenance

Savitar uses provenance logs to make the clean-room charter operational.

## Rules

1. Create a new entry whenever feature work depends on public docs, parity targets, or third-party product behavior.
2. Keep entries short and factual.
3. Record what informed the work and what was explicitly not copied.
4. Treat provenance as part of the feature definition, not post-hoc paperwork.

Use `template.md` as the starting point.
